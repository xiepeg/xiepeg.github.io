MyBadis初识

一、mybatis概念

Mybatis是Java持久层的一个Java框架，主要用于数据库的操作。相比于JDBC，它支 持定制化SQL、存储过程以及高级映射，避免了几乎所有的JDBC代码和手动设置参数以及获取结果集，简化数据库的各种操作。

3、配置主配置文件SqlMapConfig.xml

在目录resources下创建一个文件SqlMapConfig.xml，用于配置数据库连接。

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<!--mybatis的主配置文件-->
<configuration>
    <!-- 这里写配置内容 -->
    <!--如果定义了外部配置文件，如数据库连接的properties文件，也可直接导入配置文件
        resource属性：用于指定配置文件位置，按照类路径方式来写
        URL属性：按照url方式来定位配置文件位置
        -->
    <!--<properties resource="resources下的路径"></properties>-->
    <environments default="mysql">
        <!--配置环境-->
        <environment id="mysql">
            <!--配置事务类型-->
            <transactionManager type="JDBC"></transactionManager>
            <!--配置数据源-->
                <!--type表示配置连接池的方式，
                    POOLED:传统的的规范的连接池思想
                    UNPOOLED: 未采用池的思想，传统连接
                    JNDI:采用服务器提供的JNDI技术实现-->
            <dataSource type="POOLED">
            <!--配置连接池数据库的四个基本信息-->
                <property name="driver" value="com.mysql.cj.jdbc.Driver" />
                <property name="url" value="jdbc:mysql://localhost:3306/user_data?serverTimezone=UTC"/>
                <property name="username" value="root"/>
                <property name="password" value="aaa888aaa"/>
            </dataSource>
        </environment>
    </environments>
    <!--指定映射配置文件位置-->
    <mappers>
        <mapper resource="com/Mybatis/dao/UserDao.xml"/>
    </mappers>
</configuration>

定义了配置文件便能连接我们的指定的数据库User_data这里，以连接池的方式来创建。显然相对于传统的JDBC方式连接数据库，这里简化了很多操作。

注意：在这里我们指定了映射配置文件的位置，映射配置文件对应着对指定持久层接口的各种操作，这里定义的是UserDao.xml文件，将在后面步骤给出具体定义。

4、定义与数据库对应的User实体类

在src/main/java下创建包com.Mybatis.mainclass, 并在mainclass下定义User类并生成对应的getter和setter以及toString()方法：

注意：这里User的属性名定义为和数据库中的属性名一致，是为了便于后面类的封装。
package com.Mybatis.mainclass;
import java.io.Serializable;
import java.util.Date;
public class User implements Serializable {
    private Integer id;
    private String username;
    private Date birthday;
    private String sex;
    private String address;

    public Integer getId() {
        return id; }
    public String getUsername() {
        return username;
    }
    public Date getBirthday() {
        return birthday;
    }
    public String getSex() {
        return sex;
    }
    public String getAddress() {
        return address;
    }
    public void setId(Integer id) {
        this.id = id;
    }
    public void setUsername(String username) {
        this.username = username;
    }
    public void setBirthday(Date birthday) {
        this.birthday = birthday;
    }
    public void setSex(String sex) {
        this.sex = sex;
    }
    public void setAddress(String address) {
        this.address = address;
    }
    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", username='" + username + '\'' +
                ", birthday=" + birthday +
                ", sex='" + sex + '\'' +
                ", address='" + address + '\'' +
                '}';
    }
}

5、编写持久层接口 UserDao

同样在src/main/java下创建包com.Mybatis.dao,在dao包下创建用于操作数据库的接口UserDao，在这里我们不妨定义了一个方法用于查询所有User的信息：

package com.Mybatis.dao;
import com.Mybatis.mainclass.User;
import java.util.List;

public interface UserDao {
    /**
     * 查询所有所有用户信息
     * @return
     */
    List<User> findAll();
}

6、定义持久层接口UserDao的映射文件

在第三步中我们创建了映射文件UserDao.xml文件

注意的是在创建映射配置文件时，需要让该文件在main目录中的java和resources下具有相同的目录结构：

UserDao.xml文件：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.Mybatis.dao.UserDao">
    <select id="findAll" resultType="com.Mybatis.mainclass.User">
        select * from user;
     </select>

在该文件中，首先规定了名空间，也就是文件将映射到的具体接口，接着我们使用select标签，表示执行的是查询操作，UserDao中，我们定义的查询操作返回值为User集合，所以这里我们要规定返回的类型com.Mybatis.mainclass.User。

7、编写测试类

在src/test/java/com/test下定义类UserDaoTest用于测试是否成功：

public class UserDaoTest {
    @Test
    public void testfindAll() throws IOException {
        InputStream is= Resources.getResourceAsStream("SqlMapConfig.xml");
        SqlSessionFactory factory=new SqlSessionFactoryBuilder().build(is);
        SqlSession session=factory.openSession();
        UserDao dao=session.getMapper(UserDao.class);
        List<User> lst=dao.findAll();
        for (User user:lst){
            System.out.println(user);
        }
    }
}
